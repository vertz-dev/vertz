## Why this post exists

This post is a development fixture. Phase 1 of issue [#2947](https://github.com/vertz-dev/vertz/issues/2947) shipped the MDX infrastructure that takes an `.mdx` file on disk and turns it into a reachable URL under `/blog/<slug>`. Phase 2 wraps it in a proper reading surface: sticky Nav, reused Footer, 3-column desktop grid, typographic prose, a reading-progress bar, and a table of contents.

Subsequent phases (3–7) add the list page, tier-1 and tier-2 MDX components, SEO, and the first real post.

## A short list

- Posts live under `packages/landing/content/blog/`.
- Each file is compiled once via `@vertz/mdx` into an HTML string.
- A generated manifest wires every post into the landing bundle.

<Callout type="warn" title="Heads up">
  Phase 4 ships MDX custom components as build-time renderers — no runtime JS required.
</Callout>
