## CTO Vision: Full-Stack Unification
- vertz should be ONE package — UI + API + DB, not 4 separate packages
- Entity-aware UI: components should know about data models, type-safely
- Next.js /api is the anti-pattern — untyped, no modules, doesn't scale
- vertz's module system is the differentiator — real architecture for APIs
- Open question: does single-package work when users also build external APIs?
- Blitz.js tried zero-API and struggled — research what went wrong
- Discussion doc: `plans/discussions/full-stack-unification.md`
- Needs Josh research + team discussion before any code changes
