# Blog — Design Doc

**Issue:** [#2947](https://github.com/vertz-dev/vertz/issues/2947)
**Status:** Planning → Ready for Phase 1
**Date:** 2026-04-22

---

## Summary

Dogfood the Vertz stack to ship `vertz.dev/blog`. Posts are authored as `.mdx` files in the repo; the blog is served as additive routes under the existing `packages/landing` worker (no new package, no new deploy target). Dark-only, reuses landing Nav / Footer / theme.

## Why Dogfood

- Marketing narrative — "the Vertz blog runs on Vertz"
- Stress-tests `@vertz/mdx` with real long-form content
- Zero lock-in (posts are `.mdx`, portable to anywhere)
- Zero extra infra (same Cloudflare worker as landing)
- Rejects Mintlify (docs-optimized, no RSS, no listing), rejects Astro / Next (competitor stacks undermine the dogfood narrative)

## Manifesto Alignment

- **Agent-native:** `.mdx` with typed frontmatter parseable by agents; posts are reviewable as code
- **One tool, one mental model:** reuses `@vertz/ui`, `@vertz/mdx`, `@vertz/theme-shadcn` — no new stack
- **If it builds, it works:** posts live in the repo, CI validates render, no separate CMS
- **Rejected alternatives:** Mintlify (docs-first, no RSS/listing), Astro/Next (competitor stacks)

## Non-Goals (v1)

- Comments (Disqus/Giscus)
- Full-text search (reuse docs search in a later release)
- Newsletter / email subscription
- i18n
- Dedicated `/blog/tag/<tag>` routes (client-side filter only on `/blog`)
- Light mode toggle (dark-only, matches landing)
- `TypeFlow`, `VertzPlayground`, `SchemaVisualizer`, `Mermaid` (Tier 3 components — deferred)

## Unknowns

None identified.

## POC Results

None required — `@vertz/mdx` is already published and used in `packages/docs`. The blog reuses proven infra end-to-end.

## Type Flow Map

```
content/blog/*.mdx (frontmatter)
  ↓ parse
load-posts.ts → PostMeta[]
  ↓ type flows to:
  ├─ BlogListPage        (grid card rendering)
  ├─ BlogPostLayout      (post header rendering)
  ├─ rss-feed.ts         (RSS entry generation)
  ├─ sitemap.ts          (URL list)
  └─ og-cover.ts         (fallback image generation)
```

Single `PostMeta` type, consumed by five callers. Type test (`.test-d.ts`) proves frontmatter → `PostMeta` inference.

## MDX Components Ship List

### Tier 1 — Essential (Phase 4)
**HTML overrides:** `h2`–`h4`, `p`, `a`, `ul`/`ol`/`li`, `blockquote`, `hr`, `table`, inline `code`.
**Code block:** Shiki `vitesse-dark` with `title="..."`, `{1,3-5}` highlight, `showLineNumbers`, `diff`, `wrap` meta; copy button; language label; horizontal scroll.
**Custom:** `Callout` (♻️ docs), `Figure` (🆕), `CodeGroup` + `Tab` (♻️ docs), `Steps` + `Step` (♻️ docs).

### Tier 2 — Differentiator (Phase 6)
`Terminal`, `FileTree`, `Compare`, `Keyboard`, `Badge`, `Tweet` (SSR oEmbed, no runtime JS).

### Tier 3 — Deferred
`TypeFlow`, `VertzPlayground`, `SchemaVisualizer`, `Mermaid`, `VideoEmbed`, `Accordion` (♻️ docs if needed).

## Theme

- Palette: zinc (same as landing)
- Shiki theme: `vitesse-dark` — reversible single-line config in `blog/mdx/shiki-config.ts`
- Fonts: DM Sans (body, 17px, line-height 1.7), DM Serif Display (h1 only), JetBrains Mono (code) — all loaded by landing
- Cover fallback: when frontmatter omits `cover`, auto-generate via `scripts/generate-og.ts` using `title + tag + gradient zinc→accent`

## Layout Invariants

| Constraint | Value |
|---|---|
| Post body max-width | `640px` (68ch) |
| Code breakout max-width | `800px` (negative horizontal margin in post body at `>=1024px`) |
| TOC visibility | sticky right at `>=1024px`, hidden below |
| List grid | 2-col at `>=768px`, 1-col below |
| Cover aspect | `16:9` with `rounded-lg` |
| Scroll margin on headings | `80px` (accounts for sticky Nav) |
| Nav / Footer | reused from landing; no `HeroGlow` on `/blog/*` routes |

## Edge Cases (known-to-break, must handle)

| Case | Handling |
|---|---|
| Long title (>80 chars) | `text-wrap: balance`, 1rem bottom margin |
| Code overflow mobile | `overflow-x: auto` + touch scroll + shadow edge hint |
| Image without dims | `Figure` requires `width`/`height` props (type-enforced) |
| External link ambiguity | `a[href^="http"]:not([href*="vertz.dev"])` gets `ExternalLink` icon |
| Tweet embed load failure | SSR oEmbed at build time, fallback to plain link |
| Post without cover | Auto-generated OG fills the card |
